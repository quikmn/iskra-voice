Outbound webhooks fire a POST request to any URL you specify every time a message is sent. Leave the channel ID blank to receive all channels, or scope it to one. Use them to forward chat to Slack, trigger CI builds, or log activity in any external system. Inbound webhooks give you a secret POST URL you call from any external service — GitHub Actions, cron jobs, monitoring tools — to push a message into a specific channel. Each inbound hook targets one channel and uses a unique secret token. Both types work with anything that can send HTTP.

Bots connect over WebSocket exactly like regular clients but authenticate with a named token instead of a password. After auth, a bot receives every server broadcast in real time — new messages, reactions, voice state changes, status updates, and more. It can post messages to any channel using a SEND_CHAT action. What data leaves your device

Honest, complete breakdown. Last updated: May 2026.

What "zero telemetry" actually means

Iskra contains no analytics SDK, no crash reporter, no usage tracking, and no feature telemetry. The app never contacts Anthropic, any ad network, or any Iskra-controlled server unless you explicitly trigger an action. No session data, no event logs, nothing.

When you connect to a server

The following is sent to the server you connect to — which is operated by whoever runs that server, not by Iskra:

  • Your alias (display name)
  • Every message you send in text channels
  • Your voice audio while your mic is transmitting
  • Your avatar image when you upload one (stored on the server)
  • Files and images you share in chat
  • Your presence and voice state — which channel you're in, muted/deafened status
  • Reactions you send
  • Direct messages between server members (non-IskraID DMs) — stored as plaintext on the server. The server operator can read them. IskraID DMs are handled separately and are E2E encrypted (see below).
  • A machine GUID — a hardware-derived identifier used to enforce bans. It is not linked to any account and is never sent to Iskra.

You choose which servers to connect to and therefore which server operators to trust. Iskra has no visibility into any of this.

When you use IskraID (optional)

IskraID is opt-in. If you don't create an account, none of this applies. If you do, the following is sent to the IskraID relay:

  • Your email address (for account verification only — not shared or sold)
  • Your IskraID alias and avatar
  • Your friends list
  • Your public encryption key (used to establish E2E encrypted DMs)
  • Direct messages you send — these are end-to-end encrypted on your device before leaving. The relay forwards ciphertext and cannot read the content.

What never leaves your device

  • ✓ Server passwords — stored locally only
  • ✓ Local aliases — stored locally; only sent to a server when you connect to it
  • ✓ Per-server nicknames — stored locally only
  • ✓ Appearance settings and themes
  • ✓ Your full client configuration
  • ✓ Anything you type but don't send

No third parties, ever

Iskra does not use Cloudflare, AWS, Google, or any other cloud service to process your voice or messages. The only external service involved is the IskraID relay if you opt into IskraID, and GIF search which uses Tenor (your search query is sent to Tenor when you open the GIF picker).
