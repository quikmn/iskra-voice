What's in the box

A full platform.
No feature tiers.

Channels, roles, DMs, search, screenshare, soundboard, bots, audit log, per-channel E2E encryption. All of it ships in one zip, free, no subscription required.

Voice Channels

Push-to-talk or voice activation. WebRTC peer-to-peer audio with STUN/TURN for NAT traversal. No third-party audio servers, no routing through a cloud.

Text Channels

Markdown, image embeds, file uploads, reactions, edits, deletes, pinning, slow mode, KQL-style search. Messages stored as plain .jsonl files on disk.

Roles & Auth

Five role tiers: guest, member, trusted, admin, owner. Three auth modes: open, registered-plus-guests, or verified-only. HWID-based local identity.

Direct Messages & Calls

Private one-on-one conversations with read receipts. Right-click any user to start a voice or video call, share your screen, or open a DM thread.

Screenshare & Soundboard

Share your screen or webcam in any voice channel. Play audio clips from a soundboard. No extra software, no plugins.

Per-channel E2E Encryption

Flag individual channels as end-to-end encrypted. Messages are encrypted client-side before leaving your machine. Server never sees plaintext.

Audit Log & Backups

Every admin action is logged. Download a full server backup from the client - history, roles, bans, avatars, uploads - as a single ZIP.

Webhooks & Bots

Post to any text channel via HTTP webhook. Bots connect as regular clients. No rate limits, no approval process, no API key requests.

Multi-server & Themes

Connect to multiple servers at once from one client window. Seven built-in themes including Catppuccin, Nord, Tokyo Night, and Gruvbox.

Use cases

Who uses Iskra
and for what.

Iskra does not try to be everything to everyone. It is a voice and text server you run yourself. Here is where it fits well.

🎮

Gaming friend groups

A small group that plays together regularly and wants voice chat without giving Discord access to their microphone and conversation history. Server runs on whoever's PC, friends connect, done.

Best with: Mode 1 (LAN) or Mode 2 (port forward)
🖥️

LAN parties

Everyone is on the same network anyway. No port forwarding, no internet required. Spin up the server on the host machine, share the local IP, and every person at the table can connect and talk in under a minute.

Best with: Mode 1 (LAN only)
🛡️

Game clans and communities

Persistent server with multiple channels, channel categories, role hierarchy, moderation tools, and an audit log. Ventrilo or TeamSpeak veterans will recognize the model. The UI is just more modern.

Best with: Mode 4 or 5 (VPS)
🔒

Private team communications

A freelance team or small company that does not want their conversations on someone else's servers. Self-host on a VPS, restrict to registered users only, enable E2E encryption on sensitive channels.

Best with: Mode 5 (VPS + nginx + SSL)
📡

Ventrilo and TeamSpeak migration

Same mental model as classic voice servers - channels, roles, PTT. The setup is just as fast, the data stays on your machine, and you do not need a paid license or a third-party host.

Best with: Mode 2 or 4 depending on uptime needs
📺

Content creators and streamers

Soundboard built in. Screenshare and camera in voice channels. Right-click video calls with co-hosts or guests. No Nitro subscription, no feature wall, no terms of service that changes what you can do.

Best with: Mode 2 or 5
Setup guide

Seven ways to run it.
Pick yours.

Each mode has different trade-offs. Read the "what doesn't work" column before committing to a setup. Most people start with Mode 1 or 2 and never need anything else.

1

Local network (LAN only)

Server on your PC, friends on the same network connect by your local IP. Zero router changes, zero external dependencies. Ideal for LAN parties, home networks, and office setups where everyone is physically on the same connection.

Easiest No internet needed Windows
Setup steps
  1. Download iskra_server.exe from the GitHub releases page.
  2. Create a folder for your server (e.g. C:\iskra\world1\) and place a server.json inside it (see config snippet below).
  3. Run iskra_server.exe from that folder. The console will say it's listening on port 8080.
  4. Find your local IP: open a terminal and run ipconfig. Look for "IPv4 Address" - something like 192.168.1.50.
  5. If Windows Firewall prompts you, allow iskra_server through.
  6. Share 192.168.1.50:8080 with friends on the same network. They enter it in the client and connect.
// server.json - place next to iskra_server.exe { "Settings": { "ServerName": "My Server", "Port": 8080, "AdminPassword": "change-this", "AuthMode": "open" }, "Channels": [ { "Id": "v_lobby", "Name": "Lobby", "Type": "Voice" }, { "Id": "t_gen", "Name": "general", "Type": "Text" } ] }
What works / what doesn't
  • ✓ All features without exception
  • ✓ Voice is direct P2P on the LAN - lowest possible latency
  • ✓ No STUN or TURN required
  • ✓ No router changes, no firewall rules
  • ✗ Friends outside your local network cannot connect
  • ✗ Not accessible over the internet in any way
2

Home PC, port forwarded - friends anywhere

One TCP port open on your router. Friends connect via your public IP from anywhere on the internet. This is the "spin it up in five minutes" setup for small groups. Voice uses WebRTC with Google's STUN server by default, which handles NAT traversal for the vast majority of home connections.

Easy Router access needed Dynamic IP caveat
Setup steps
  1. Get Mode 1 working on your LAN first so you know the server starts correctly.
  2. Log in to your router admin panel (usually 192.168.1.1 or 192.168.0.1).
  3. Find Port Forwarding. Add a rule: External TCP port 8080 forwarded to your PC's local IP on port 8080.
  4. Find your public IP at ipv4.icanhazip.com.
  5. Share your_public_ip:8080 with friends. They enter it in the client and connect.
  6. Optional: set up a free dynamic DNS at DuckDNS so your address does not change when your ISP rotates your IP.
CGNAT check: If your router's WAN IP and your public IP are different, your ISP is giving you a shared address (carrier-grade NAT). Port forwarding will not work in that case. Contact your ISP or use Mode 6 or 7 instead.
What works / what doesn't
  • ✓ All features
  • ✓ Voice works for most users - STUN handles typical home NAT
  • ✓ No extra services to run
  • ! Voice may fail between two users who are both behind strict symmetric NAT - uncommon but possible. Fix with Mode 3.
  • ✗ CGNAT: port forwarding has no effect
  • ✗ Your home IP is public to anyone you share the address with
  • ✗ No SSL - web client users cannot connect without a reverse proxy
3

Home PC, port forwarded - own coturn server

Same as Mode 2 but voice relay runs through a TURN server you control. Fixes voice for users behind strict symmetric NAT and removes the dependency on third-party STUN. Requires a separate machine with a public IP to run coturn - a $4-6/mo VPS is enough.

Intermediate Requires VPS for TURN Full independence
Setup steps
  1. Get a VPS (Hetzner, DigitalOcean, Vultr - smallest tier works). Install Ubuntu or Debian.
  2. Install coturn: apt install coturn.
  3. Edit /etc/turnserver.conf - see config snippet below.
  4. Open UDP and TCP port 3478 in your VPS firewall / security group.
  5. Start coturn: systemctl enable --now coturn.
  6. In your Iskra server.json, add the TurnUrls, TurnUsername, and TurnCredential fields (see snippet).
  7. Restart iskra_server. The TURN config is sent to clients on connect automatically.
# /etc/turnserver.conf (minimal) listening-port=3478 fingerprint lt-cred-mech realm=turn.your-domain.com user=iskra:your-secret-password no-stdout-log
// server.json - add inside "Settings" "TurnUrls": ["turn:turn.your-domain.com:3478"], "TurnUsername": "iskra", "TurnCredential": "your-secret-password"
What works / what doesn't
  • ✓ All features
  • ✓ Voice works for all NAT types including strict symmetric
  • ✓ No dependency on any third-party infrastructure
  • ✓ coturn is free and open source
  • ✗ More moving parts - coturn needs to stay running
  • ✗ You still need port forwarding on the home router for the Iskra server itself
  • ✗ VPS costs money - usually $4-6/mo
4

VPS - always on, public IP, no port forwarding

Run iskra_server directly on a cloud VPS. The VPS has a public IP by default, so no router changes are needed. Server stays up 24/7 even when your home PC is off. Good for communities that need reliable uptime. The smallest VPS tier from any provider (1 CPU, 1 GB RAM) handles dozens of concurrent users with room to spare.

Intermediate ~$5/mo VPS 24/7 uptime
Setup steps
  1. Provision a VPS. Ubuntu 22.04 or Windows Server both work. Hetzner CX22 (~$4/mo) or DigitalOcean Basic Droplet are common choices.
  2. Open TCP port 8080 (and any other ports you plan to use) in the VPS firewall or security group.
  3. Linux: install .NET 8 runtime (apt install dotnet-runtime-8.0 after adding the Microsoft package repo). Copy iskra_server to the VPS via SCP or rsync.
  4. Windows VPS: copy iskra_server.exe, install .NET 8 Desktop Runtime if prompted, and run it.
  5. Create server.json in the server folder. Start iskra_server.
  6. On Linux, create a systemd service so the server restarts on reboot - see snippet below.
  7. Friends connect to your_vps_ip:8080. No port forwarding step required.
# /etc/systemd/system/iskra.service [Unit] Description=Iskra Voice Server After=network.target [Service] WorkingDirectory=/opt/iskra/world1 ExecStart=/usr/bin/dotnet /opt/iskra/iskra_server.dll Restart=always RestartSec=5 User=iskra [Install] WantedBy=multi-user.target
What works / what doesn't
  • ✓ All features
  • ✓ 24/7 uptime - server stays on when your PC doesn't
  • ✓ Public IP built in - no port forwarding
  • ✓ Your home IP is never exposed
  • ! Voice still uses default STUN. Add TURN config (Mode 3) for strict NAT users.
  • ✗ No SSL/TLS on this mode - web client needs HTTPS (use Mode 5)
  • ✗ Costs money monthly
  • ✗ Server binary is a Windows .exe - on Linux you run it via the .NET runtime, not Wine
5

VPS + nginx reverse proxy - domain, SSL, web client

The recommended setup for any public-facing server. nginx sits in front of Iskra and handles TLS termination. Friends connect to a proper domain over WSS. The web client works. Let's Encrypt makes the SSL cert free.

Advanced Domain required Free SSL via certbot
Setup steps
  1. Start with Mode 4 - get iskra_server running on your VPS on port 8080 (localhost only is fine; nginx will proxy).
  2. Point a domain or subdomain to your VPS IP via an A record (e.g. talk.your-domain.com).
  3. Install nginx: apt install nginx.
  4. Install certbot: apt install certbot python3-certbot-nginx.
  5. Get a certificate: certbot --nginx -d talk.your-domain.com. Follow the prompts.
  6. Create an nginx site config as shown below and reload nginx.
  7. Friends now connect to talk.your-domain.com (no port needed). Web client users can join from any browser.
# /etc/nginx/sites-available/iskra server { listen 80; server_name talk.your-domain.com; return 301 https://$host$request_uri; } server { listen 443 ssl; server_name talk.your-domain.com; ssl_certificate /etc/letsencrypt/live/talk.your-domain.com/fullchain.pem; ssl_certificate_key /etc/letsencrypt/live/talk.your-domain.com/privkey.pem; location / { proxy_pass http://127.0.0.1:8080; proxy_http_version 1.1; proxy_set_header Upgrade $http_upgrade; proxy_set_header Connection "upgrade"; proxy_set_header Host $host; proxy_read_timeout 86400s; proxy_send_timeout 86400s; } }
What works / what doesn't
  • ✓ All features
  • ✓ Clean domain URL, no port numbers to share
  • ✓ SSL - web client connects from any browser
  • ✓ Certbot auto-renews the cert every 90 days
  • ✓ Iskra server only listens on localhost - not directly exposed
  • ! Voice still uses STUN by default. Add TURN for bulletproof NAT traversal (see Mode 3 TURN config).
  • ✗ Most complex setup - more things can go wrong
  • ✗ Requires domain ownership and DNS propagation time (~5 min on Cloudflare, up to 48h elsewhere)
6

Cloudflare Tunnel - no open ports, no exposed IP

If you cannot or will not open ports at home (CGNAT, restrictive ISP, apartment building, etc.), Cloudflare Tunnel creates an outbound-only connection from your PC to Cloudflare's network. Friends connect to a Cloudflare URL. Your home IP is never shared. Requires a free Cloudflare account and a domain.

Intermediate Free (Cloudflare) CGNAT-safe
Setup steps
  1. Get iskra_server running locally (Mode 1) and confirm it works.
  2. Create a free Cloudflare account. Add your domain to Cloudflare and point its nameservers.
  3. Install cloudflared on your PC (download from Cloudflare's GitHub releases).
  4. Authenticate: cloudflared tunnel login.
  5. Create a tunnel: cloudflared tunnel create iskra. Note the tunnel ID.
  6. Create a config file pointing the tunnel at localhost:8080 and create a DNS CNAME in Cloudflare pointing to the tunnel.
  7. Run: cloudflared tunnel run iskra. Friends connect to the CNAME hostname.
# ~/.cloudflared/config.yml tunnel: <your-tunnel-id> credentials-file: /root/.cloudflared/<id>.json ingress: - hostname: talk.your-domain.com service: http://localhost:8080 - service: http_status:404
Voice limitation: Cloudflare Tunnel is TCP-only. The WebSocket signaling channel works fine. But WebRTC voice data travels over UDP, which bypasses the Cloudflare tunnel entirely. Voice will only work if both parties can do direct P2P (STUN) or if you configure a TURN server (Mode 3 TURN steps still apply here).
What works / what doesn't
  • ✓ Text channels, DMs, search, file uploads
  • ✓ WebSocket signaling (voice channel join/leave, peer negotiation)
  • ✓ Works from behind CGNAT - no port forwarding at all
  • ✓ Your home IP is never exposed to clients
  • ✓ Free tier is enough for most uses
  • ! Voice requires STUN P2P to work - add a TURN server if P2P fails
  • ✗ Voice data (UDP) does not go through the tunnel - TURN config needed for strict NAT users
  • ✗ Requires a domain name
7

VPN overlay (Tailscale / ZeroTier) - private network

Create a virtual LAN that friends join. Iskra sees it as a local network connection, so it behaves exactly like Mode 1. No port forwarding, no public IP exposure, no CGNAT issues, no TURN required. Voice is peer-to-peer directly over the VPN. Every friend must install and join your VPN, so this works best for tight-knit groups, not open servers.

Easy once set up Free (Tailscale up to 100 devices) All users must join VPN
Setup steps (Tailscale)
  1. Sign up at tailscale.com. Install Tailscale on your PC and log in.
  2. Start iskra_server normally (Mode 1 setup). No changes to server.json needed.
  3. Find your Tailscale IP in the Tailscale app - it will be in the 100.x.x.x range.
  4. Each friend installs Tailscale, logs in, and you share an invite link to join your tailnet (Tailscale admin panel).
  5. Friends connect to your Tailscale IP on port 8080 from the Iskra client. Done.
ZeroTier: Works the same way. Install ZeroTier on all machines, create a network at my.zerotier.com, approve each member, connect using the ZeroTier IP. Slightly more manual than Tailscale but fully self-hostable.
What works / what doesn't
  • ✓ All features without exception
  • ✓ Voice is direct P2P over the VPN - no TURN needed
  • ✓ Works from behind CGNAT
  • ✓ Your home IP is never shared
  • ✓ No open ports on your router
  • ✗ Every friend must install Tailscale or ZeroTier - no public access
  • ✗ Not suitable for open servers or anyone you can't onboard to the VPN

Quick start

Up in three steps.

No installer, no cloud setup, no account creation. Unzip, write a config, run.

1

Run the server

Create a folder, drop in a server.json, run iskra_server.exe. It is listening in seconds.

2

Share the address

Forward one TCP port on your router. Give friends your IP and port. No domain, no SSL cert, no dashboard required.

3

Connect and talk

Friends unzip the client, enter your address, pick a name, and they are in. Voice, text, files - all of it works immediately.
